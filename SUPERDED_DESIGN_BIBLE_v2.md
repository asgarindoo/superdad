# SuperDed Design Bible v2

> **Version:** 2.0
> **Purpose:** Product Design Guidelines
> **Design Philosophy:** Clean • Playful • Cozy • Modern

---

# Product Vision

**SuperDed** is a virtual home where people can create a customizable father figure who listens, remembers, grows, and shares meaningful conversations.

SuperDed is **not** a chatbot.

SuperDed is **not** an AI assistant.

SuperDed is a **virtual home experience**.

When users open SuperDed, they should feel like:

> "I want to step into this world."

—not—

> "I want to try another AI."

---

# Color System

Forget the heavy brown palette.

The UI should feel bright, fresh, and warm.

## Neutral Colors

| Usage | Color |
|--------|--------|
| Background | `#FCFCFA` |
| Surface | `#F6F7F3` |
| Card | `#FFFFFF` |
| Primary Text | `#202020` |
| Secondary Text | `#707070` |
| Border | `#ECECE8` |

---

## Accent Colors

### Sage

`#72B38A`

### Terracotta

`#E97B57`

### Dusty Blue

`#7FAFD9`

### Soft Mustard

`#E7C25B`

### Coral

`#F08C83`

Accent colors should be used sparingly.

No purple AI gradients.

No neon colors.

# Typography

The typography should feel friendly and timeless.

## Heading

- Outfit
- Plus Jakarta Sans

## Body

- Plus Jakarta Sans
- Manrope

Avoid:

- Instrument Serif
- Space Grotesk
- Satoshi
- AI startup typography combinations

---

# Shape Language

Keep shapes soft but structured.

| Component | Radius |
|------------|---------|
| Button | 14px |
| Card | 16px |
| Input | 12px |
| Modal | 20px |

Avoid oversized pill buttons.

---

# Shadows

Use soft shadows.

```css
box-shadow: 0 10px 35px rgba(0,0,0,.05);
```

Avoid glow effects.

---

# Borders

```css
1px solid #ECECE8;
```

Minimal and subtle.

---

# Layout

- Fullscreen
- Responsive
- Desktop-first
- Mobile-friendly
- Large whitespace
- Asymmetrical layouts
- Editorial composition

Never feel like a dashboard.

Every section should breathe.

# Motion

Animations should be:

- Calm
- Smooth
- Natural
- Elegant

Hover:

- Slight lift
- Slight shadow increase

Scroll:

- Fade
- Scale
- Soft parallax

Avoid flashy animations.

---

# Icons

Use:

- Lucide Icons

Style:

- Thin stroke
- Minimal
- Friendly

---

# Hero Section

The hero is the heart of the product.

## Do NOT

- Put text on the left and character on the right
- Show a chatbot interface
- Fill the hero with UI
- Add dashboard widgets
- Add analytics cards

---

## Goal

Users should immediately understand:

> "I can create my own father."

The hero should focus on:

- A beautiful room
- The father
- Atmosphere
- Emotion

Minimal interface.

---

# Interactive Hero

Show customization possibilities.

Examples:

Room

- 🏡 Scandinavian
- 🌿 Japanese
- 🌊 Coastal
- 🔥 Cabin

Character

- 👕 Outfit
- 🎤 Voice
- 😊 Personality
- 💬 Conversation Style

Changing these should update the preview.

---

# Feature Section

Avoid:

Icon + Paragraph layouts.

Instead, use large visual storytelling cards.

Example:

```
🧠

He Remembers
Every Story

Illustration
```

# Character Creator

Inspired by:

- The Sims
- Nintendo Mii
- Pixar

Not:

- Forms
- Accordions
- Settings pages

The character should occupy around 70% of the screen.

---

# Conversation Room

This is the most important screen.

Approximately 80% of the viewport should be the room.

Minimal floating controls only.

Examples:

- 🎤 Voice
- 💬 Chat
- 📷 Camera
- ⚙ Settings

The room should always remain the focus.

# Design Rules

- Never look like AI software.
- Never look like a chatbot.
- Never look like a dashboard.
- Never use AI-style purple gradients.
- Never overload screens with cards.
- Let whitespace become part of the design.
- The room is always more important than the interface.
- The father is always more important than the buttons.
- Every interaction should feel calm, playful, and emotionally comforting.
